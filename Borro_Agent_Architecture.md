# Borro Agent - Architecture & PRD

## 1. Product Vision (О проекте)
**Название:** Borro Agent
**Суть продукта:** Автономный non-custodial агент, который круглосуточно управляет кредитной позицией пользователя (Obligation) в Kamino Finance. 
**Ценность:** Защита от дефолта на основе **предиктивного**, а не реактивного анализа. В то время как обычные боты продают ваш залог только когда он уже обесценился (ликвидация), Borro Agent анализирует рыночный "страх" и волатильность, чтобы снизить кредитное плечо (LTV) **до** обвала рынка.

---

## 2. Полная логика работы (User Flow & System Logic)

1. **Инициализация (Onboarding):**
   * Пользователь заходит в веб-интерфейс, подключает кошелек (Phantom/Backpack).
   * Видит свою текущую позицию в Kamino (залог SOL, долг USDC, текущий Health Factor).
   * Нажимает "Enable AI Guard". Подписывает транзакцию, делегирующую агенту определенные права (или использует специальный PDA-смарт-контракт Vault, но для MVP достаточно дать агенту право на repay/withdraw).

2. **Мониторинг (Cron-Job / Daemon):**
   * Скрипт (сервер) раз в 5 минут собирает метрики: 
     - *On-chain:* Текущий `LTV` позиции из Kamino SDK.
     - *Off-chain:* Историческая волатильность актива (Binance/CoinGecko API), индекс страха и жадности, опционально — парсинг новостей.

3. **Принятие решения (AI Brain - OpenAI GPT-4o):**
   * Данные упаковываются в промпт: *"You are an auto-deleveraging AI for Kamino DeFi protocol. Current volatility is high..."*
   * Модель с помощью Function Calling (или возврата JSON) отдает вердикт: 
     `{ "target_ltv": 0.35, "action": "DELEVERAGE", "reason": "High selling pressure in SOL detected" }`

4. **Экзекьюшн (Ребалансировка позиции):**
   Если текущий LTV позиции 0.60, а AI требует 0.35, бот собирает транзакцию:
   * **Шаг A:** `Kamino.withdraw()` — изымает часть залога (например, 10 SOL).
   * **Шаг B:** `Jupiter.swap()` — меняет эти 10 SOL на $1400 USDC.
   * **Шаг C:** `Kamino.repay()` — гасит $1400 USDC долга.
   *(Для MVP хакатона все 3 шага можно упаковать в одну транзакцию, либо, если это сложно, разделить на три последовательные).*

## 3. Требования (MVP для Хакатона)

Чтобы жюри начислило максимум баллов (по критериям Technical Implementation и AI + Blockchain):

* **[Required] On-chain интеграция:** Бот должен уметь хотя бы читать статус аккаунта из Kamino (через `Kamino.getObligation`) в сети `Devnet` или `Mainnet`. Обязательно должна происходить настоящая транзакция в блокчейне (хотя бы `repay` тестовыми USDC в Devnet).
* **[Required] AI Связка:** Логика изменения `target_ltv` не должна быть захардкожена. В логах (терминале) должно быть четко видно, как AI получает данные и возвращает решение на основе промпта.
* **[Mocked (Опционально)]:** Если собирать сложную Swap-транзакцию с Jupiter'ом тяжело, для MVP можно **замокать Шаг B (Jupiter Swap)** — то есть бот просто берет заранее лежащие на кошельке запасные тестовые USDC и делает Repay на нужную сумму. Главное — показать, что AI инициировал транзакцию спасения/ребалансировки!
* **[UX/UI] Фронтенд:** Простой дашборд (React + Next.js), который показывает "Текущий LTV: 70%, Целевой LTV от ИИ: 40%, Статус: идет балансировка...".

## 4. Архитектура Решения

```text
[ Пользователь / Браузер ]
          |
  (Подписание транзакций, просмотр стаффа)
          |
[ Next.js Backend (Node.js/TypeScript) ] <----------------+
          |                                               |
  1. Cron Job Scheduler (каждые 1-5 минут)                | Off-chain данные 
          |                                               | (CoinGecko API, News API)
          |-----> [ Модуль ИИ (OpenAI API) ] <------------+
          |        (Возвращает нужный Target LTV)
          |
  2. Transaction Builder
          | (Использует @solana/web3.js и Kamino SDK)
          |
[ Solana Blockchain ]
          |
  --> Смарт-контракт Kamino Lending
  --> Jupiter Aggregator (Опционально для свапа)
```

## 5. Пошаговый план разработки (Action Plan)

**День 1: Ядро (Kamino + Node.js)**
1. Создайте пустой Node.js проект. Установите `@solana/web3.js` и `@hubbleprotocol/kamino-lending-sdk`.
2. Напишите простой скрипт (с хардкод-приватником тестового кошелька), который подключается в `Devnet`, находит ваш кредитный аккаунт (Obligation) и просто пишет в консоль: `"Текущий долг: X, Текущий залог: Y"`.

**День 2: Интеллект (OpenAI)**
3. Напишите функцию, которая имитирует падение рынка (передает в LLM плохие данные) и возвращает `target_ltv`.
4. Свяжите это: Скрипт видит реальный LTV (из Шага 1), получает Target LTV (из Шага 3) и пишет условие: `if (realLTV > targetLTV) -> console.log("Надо ребалансировать")`.

**День 3: Исполнение и UI (Транзакции)**
5. Допишите вызов транзакции `repay()` через Kamino SDK.
6. Накидайте очень простой UI (Next.js + Tailwind), где показываются данные позиции пользователя и есть кнопка "Делегировать AI".
